ที่มาและความสำคัญของโครงงาน

ปัญหาหรือโอกาสที่ต้องการแก้
ในปัจจุบัน หลายองค์กรยังคงประสบปัญหาในการบริหารจัดการการลาของพนักงานที่ซ้ำซ้อนและใช้กระชับเวลา ไม่ว่าจะเป็นการยื่นเอกสารแบบกระดาษ การรอคอยการอนุมัติที่ล่าช้า หรือการเข้าถึงข้อมูลวันลาคงเหลือที่ยากลำบาก กระบวนการเหล่านี้มีความเสี่ยงต่อการสูญหายของข้อมูลและทำให้เกิดความสับสนระหว่างบุคลากรและฝ่ายทรัพยากรบุคคล (HR) จึงเกิดโอกาสในการนำเทคโนโลยีมาปรับปรุงกระบวนการเหล่านี้ให้เป็นระบบอัตโนมัติ 

เหตุผลที่เลือกทำหัวข้อนี้
โครงงานนี้ถูกพัฒนาขึ้นเพื่อแก้ปัญหาความล่าช้าและความผิดพลาดในการจัดการการลา โดยมุ่งเน้นการสร้าง "ระบบจัดการการลาออนไลน์" ที่ทันสมัย ด้วยสถาปัตยกรรมแบบ Full-stack (React, NestJS, Prisma) ซึ่งมีประสิทธิภาพและความปลอดภัยสูง (เช่น การเชื่อมต่อ Auth Oauth2 และ Two-Factor Authentication) ทำให้สามารถลดภาระงานแอดมินและเพิ่มความสะดวกสบายให้กับผู้ใช้งานได้ในเวลาเดียวกัน

กลุ่มผู้ใช้หรือบริบทของงาน
ระบบนี้ถูกออกแบบมาเพื่อรองรับ 3 กลุ่มผู้ใช้งานหลัก:
1. พนักงานทั่วไป (Employee): ต้องการดูโควต้าวันลา, ยื่นขอลา, และติดตามสถานะการอนุมัติแบบเรียลไทม์
2. ผู้จัดการ (Manager): ต้องการดูรายการการลาของทีมงานตนเอง และทำการอนุมัติหรือปฏิเสธคำขอ
3. ผู้ดูแลระบบ/ฝ่ายทรัพยากรบุคคล (Admin/HR): ต้องการดูข้อมูลภาพรวม จัดการโควต้าของพนักงาน เปลี่ยนแปลงบทบาท และตรวจสอบการใช้งานระบบ (Audit Logs)

คุณค่าหรือประโยชน์ที่คาดว่าจะเกิด
- ความรวดเร็วและแม่นยำ: ลดเวลาในกระบวนการขอลางานและการอนุมัติ
- โปร่งใสและตรวจสอบได้: พนักงานสามารถตรวจสอบสิทธิคงเหลือของตนเองได้ตลอดเวลาผ่าน Dashboard และระบบมีการเก็บ Log ทุกครั้งที่มีการใช้งาน API
- ความปลอดภัยระดับสูง: ข้อมูลถูกปกป้องด้วยระบบ JWT Cookies และรองรับรหัสผ่านแบบ 2FA ช่วยป้องกันการโจรกรรมบัญชี

ขอบเขตเบื้องต้นและสมมติฐานสำคัญ
โครงงานนี้พัฒนาเป็น Web Application (SPA) รับรอบการใช้งานผ่านเบราว์เซอร์ โดยมีระบบป้องกันด้วยการทำ Reverse Proxy (Pangolin) ภายใต้ Environment แบบ Docker 
สมมติฐานหลักคือระบบสามารถรองรับการปรับขยายฐานข้อมูลในอนาคต (Scale-up) จาก SQLite สู่ PostgreSQL เพื่อรองรับจำนวนพนักงานในองค์กรขนาดใหญ่ได้ทันทีเมื่อนำไปใช้งานจริง
